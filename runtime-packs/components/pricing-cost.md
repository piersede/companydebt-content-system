# Component Pack: Pricing / Cost Pages

Apply 2–3 of the following. Never all. Full specs: `editorial-os/28-readability-components.md`.

## Recommended stack

**1. Cost Reality** (`.cd-callout--cost-reality`) — Primary
- Place before or after the main cost table/range section.
- State all assumptions. Name the variables that change the cost. Evidence level A or B.

**2. Do / Don't Panel** (`.cd-do-dont`) — Primary
- Use to show what affects cost positively vs. negatively (e.g. early action vs. late action).
- 3–5 rows. Not adjacent to any table already on the page.

**3. Key Takeaway** (`.cd-key-takeaway`) — Primary
- Place after the most complex pricing section (e.g. where fee structures overlap).
- 1–3 sentences. No new claims.

**CompanyDEBT View** (`.cd-callout--companydbt-view`) — Secondary
- Use where the page takes an explicit position on cost drivers that differs from industry default.

## Rhythm constraint

Max 3 components. Do / Don't Panel must not be adjacent to any existing table. Cost Reality must not repeat figures already in the table above it.
