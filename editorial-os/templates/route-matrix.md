# Company Debt Route Matrix

Use this matrix when planning briefs, outlines, and page architecture for insolvency content. Each row is a distinct persona-state with its own page family. Do not merge rows into a single page unless the page is explicitly a cross-route comparison.

## Route Families

The following are separate route families. Content should not treat them as co-equal sections within a single guide unless the page declares itself as a comparison page.

1. **Insolvent closure** (CVL) — director-initiated, orderly wind-down
2. **Forced closure** (compulsory liquidation) — creditor-initiated, court-ordered
3. **Solvent closure** (MVL) — tax-efficient extraction, all debts payable
4. **Business rescue** (CVA, administration) — company continues trading
5. **Director risk / liability** — personal exposure, disqualification, DLA, wrongful trading
6. **Strike-off vs formal process** — cost comparison, risk education

## Branch-Out Rules

When a page serves one route family but an adjacent route is relevant:

1. Use a short qualification block (2-3 sentences max)
2. State who the adjacent route applies to
3. Link to the dedicated page for that route
4. Do not expand the adjacent route into a full section

**Pattern:**
> "If your company can pay all debts in full, you likely need an MVL, not this insolvent liquidation guide. See our [MVL guide](/members-voluntary-liquidation/) for solvent closure."

> "If a creditor has already filed a winding-up petition, this CVL guide may be too late. See our [winding-up petition defence](/winding-up-petition/) page for immediate steps."

## Using This Matrix

- **At brief stage:** Identify the primary persona-state. If the brief maps to two rows, split the page.
- **At outline stage:** Check each H2 section. If a section serves a different row's persona-state, convert it to a branch-out block.
- **At review stage:** Run the persona-state purity gate. 80%+ of content must serve the declared persona-state.
