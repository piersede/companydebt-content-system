# Trust Pass Checklist (v1 — DEPRECATED)

> **DEPRECATED as of v2.1.** This checklist predates v2 governance and does not cover evidence-carrying claims, Treatment 2 caps, the two-part first-person test, AI prose fingerprints, or universal Company Debt mention rules.
> Use instead: the inline trust pass in `03-workflow-playbook.md` (Stage 6) and the pre-publish gate in `16-pre-publish-gate.md`.
> Do not use this file as a compliance reference.

Run this checklist against every draft before adversarial review.
## Source honesty
- [ ] Every verified fact has an attributable source
- [ ] Every inferred point is framed as interpretation, not raw fact
- [ ] Every editorial judgement is marked as opinion or assessment
- [ ] Every claim needing human confirmation has a placeholder, not a bluff
- [ ] No uncertainty has been flattened to sound more authoritative
## Disclosures
- [ ] Commercial relationships are disclosed where relevant
- [ ] Affiliate relationships are disclosed where relevant
- [ ] Review methodology limits are stated
- [ ] Whether the product was tested directly or assessed from public evidence is clear
## Methodology
- [ ] The page includes a methodology note (what was reviewed, how, when)
- [ ] Sources are presented in a way that increases trust, not clutter
- [ ] The last review date is included on decision-stage pages
## Byline and authorship
- [ ] The page has a named byline (or a clear reason why not)
- [ ] The author bio is short, specific, and credible
- [ ] No inflated personal branding or vague expertise claims
## Comparison tables
- [ ] Tables state what is being compared
- [ ] Criteria are defined
- [ ] No misleading completeness
- [ ] Estimated, inferred, or pending data is labelled
- [ ] Comparison data is dated
## Confidence calibration
- [ ] No claim is stated with more certainty than the evidence supports
- [ ] Borderline claims use hedging language (from what we can see, based on available evidence)
- [ ] Stale data is flagged or removed
## Proof signals
- [ ] Named author present
- [ ] Methodology note present
- [ ] Updated date present (where appropriate)
- [ ] Disclosure note present (where appropriate)
- [ ] Fit / non-fit guidance is clear
- [ ] Uncertainty labels are explicit where needed
