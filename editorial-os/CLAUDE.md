This directory contains the canonical editorial operating system.

Use this local file when work is primarily about governance, editorial rules, or workflow design.
Do not load the whole directory by default during routine drafting if runtime packs are sufficient.

## Canonical ownership

- `09-voice-governance.md`: voice, authorship, first-person discipline
- `10-evidence-governance.md`: claim support and evidence treatment
- `11-comparison-governance.md`: comparison logic, containment, disclosure
- `12-structure-governance.md`: openings, sections, endings
- `13-readability-governance.md`: formatting and readability
- `16-pre-publish-gate.md`: final gate
- `17-audience-and-persona.md`: default reader
- `18-seo-signal-governance.md`: search signal rules
- `23-prose-quality-gates.md`: YMYL prose enforcement
- `24-content-registry.md`: entity ownership, metadata, anti-cannibalisation
- `25-update-logic.md`: freshness, volatility, rerun logic
- `26-call-out-box-governance.md`: decision-critical call-out box rules
- `27-article-type-structure.md`: canonical page-type structures and decision-layer endings
- `docs/human-authorship-voice-engine.md`: the full human-authorship model

## Working rule

- Canonical governance defines quality.
- Runtime packs optimize delivery.
- If a runtime pack and canonical governance appear to conflict, canonical governance wins.
- Human-authorship markers are non-compressible. Do not thin them out.

## When editing this directory

- update the relevant canonical rule file
- keep rule ownership clear
- reflect meaningful routing changes in `runtime-packs/`
- update benchmarks if execution-layer expectations change
